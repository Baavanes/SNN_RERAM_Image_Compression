===========================================================

Purpose
-------

This text file is a copy/paste prompt pack for Codex on a fresh Ubuntu
22.04/24.04 machine. Use the prompts in order.

Goal:

1. Install all tools needed for Caravel cocotb and OpenLane/LibreLane.
2. Clone the BMsemi reference repository.
3. Create a separate demo package named BMSemi_SNN_image_comp.
4. Implement a simple image-compression and X1-storage RTL demo.
5. Verify it using Caravel cocotb.
6. Generate waveforms.
7. Run OpenLane/LibreLane hardening for user_project_wrapper.
8. Collect final GDS and signoff reports.

Reference repo:

  https://github.com/BMsemi/IMPACT_SNN_RERAM_submit

Expected image demo behavior:

  pixel[3] = 200, threshold = 128 -> compressed bit 3 = 1 -> X1[1][3] = 1
  pixel[8] = 64,  threshold = 128 -> compressed bit 8 = 0 -> X1[1][8] = 0

Expected compressed mask:

  0x0008

Expected cocotb result:

  TESTS=1 PASS=1 FAIL=0 SKIP=0

Prompt 4 - Implement Image Compression RTL
------------------------------------------

BEGIN PROMPT

Implement the simple image-compression and X1-storage RTL demo in:

  ~/bmsemi_x1_demo/BMSemi_SNN_image_comp

Create or replace:

  verilog/rtl/image_compression_storage.v
  verilog/rtl/user_project_wrapper.v

Functional requirement:

The design receives image commands from Caravel Wishbone, stores raw pixel
values in a tiny 16-pixel local buffer, threshold-compresses the block into a
16-bit mask, programs bright-pixel bits into the X1 IP, and later reads bits
back from X1.

Address map:

  0x3000_0000, word offset 0: status register
  0x3000_0004, word offset 1: command register on write, last command on read
  0x3000_0008, word offset 2: result register
  0x3000_000C, word offset 3: compressed mask debug register

Host command format:

  [31:28] opcode
  [27:24] image block number
  [23:16] pixel value
  [15:8]  threshold
  [7:4]   reserved
  [3:0]   pixel index inside 16-pixel block

Supported host opcodes:

  0x2 = LOAD_PIXEL
  0x3 = COMPRESS_BLOCK
  0x4 = READ_BIT
  0x5 = CLEAR_BLOCK
  0x6 = READ_MASK

Internal storage:

  reg [7:0]  pixel_buf [0:15]
  reg [15:0] valid_mask_r
  reg [15:0] compressed_mask_r
  reg [15:0] mask_read_r
  reg [7:0]  count_r
  reg [31:0] result_r
  reg        done_r
  reg        configured_r
  reg        error_r
  reg        compressed_ready_r

Status register format:

  [31:24] = 0x49
  [21:17] = state
  [16]    = compressed_ready
  [15]    = configured
  [14]    = error
  [13]    = done
  [12]    = busy
  [11:4]  = count
  [3:0]   = current block

State machine:

  ST_CFG_WRITE       : send three X1 config words after reset
  ST_IDLE            : wait for firmware command
  ST_COMPRESS_NEXT   : scan pixel indexes 0..15
  ST_COMPRESS_WRITE  : wait for X1 program ACK
  ST_READ_CMD        : issue X1 read command
  ST_READ_POP        : pop X1 read result
  ST_CLEAR_NEXT      : clear next X1 bit in block
  ST_CLEAR_WRITE     : wait for X1 reset ACK
  ST_MASK_NEXT       : read next X1 bit for full mask
  ST_MASK_CMD        : wait for X1 read-command ACK
  ST_MASK_POP        : collect X1 read result
  ST_DONE            : set done flag and return to idle

X1 address:

  32'h3000_0004

X1 config words sent after reset:

  32'hA203_C40F
  32'h0F03_0D43
  32'h4200_0C03

X1 command helpers:

  Program/set selected X1 cell:

    32'hC000_00FF | (row << 25) | (col << 20)

  Reset selected X1 cell:

    (row << 25) | (col << 20)

  Read selected X1 cell:

    32'h4000_0000 | (row << 25) | (col << 20)

Compression behavior:

  For index 0..15:

    if valid_mask_r[index] == 1 and pixel_buf[index] >= threshold:
        compressed_mask_r[index] = 1
        count_r++
        program X1[row=block][col=index] = 1
    else:
        compressed_mask_r[index] = 0

READ_BIT behavior:

  Read X1[row=block][col=index].
  Return the bit in result_r[0].

READ_MASK behavior:

  Read X1[row=block][col=0..15].
  Rebuild a 16-bit mask.
  Return {valid flag, block, count, mask}.

CLEAR_BLOCK behavior:

  Reset X1[row=block][col=0..15] to 0.
  Clear valid_mask_r and compressed_mask_r.

Error behavior:

  If a command is written while busy or before configured:
    result_r = 32'h8000_00EF
    error_r = 1
    done_r = 1

  If opcode is unknown:
    result_r = 32'h8000_00EE
    error_r = 1
    done_r = 1

Wrapper requirement:

  user_project_wrapper must instantiate only:

    image_compression_storage img_comp_inst

  Do not instantiate adaptive_fabric_top_tapeout or old protocol demo logic.

  Connect:

    la_data_out = {96'd0, img_debug_status}
    user_irq = img_event_irq

  Keep basic scan and analog X1 pins connected to io_in/io_out/analog_io in the
  same style as the reference repo.

After editing, run syntax checks:

  cd ~/bmsemi_x1_demo/BMSemi_SNN_image_comp
  iverilog -g2012 -I verilog/rtl \
    -o /tmp/image_comp_syntax.vvp \
    verilog/rtl/defines.v \
    verilog/rtl/Neuromorphic_X1_Beh.v \
    verilog/rtl/nvm_core_decoder.v \
    verilog/rtl/nvm_neuron_block.v \
    verilog/rtl/nvm_neuron_spike_out.v \
    verilog/rtl/nvm_synapse_matrix.v \
    verilog/rtl/nvm_neuron_core_256x64.v \
    verilog/rtl/image_compression_storage.v \
    verilog/rtl/user_project_wrapper.v

Fix all syntax errors before continuing.

END PROMPT
